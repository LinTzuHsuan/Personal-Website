Smart Chair Sensor

This is a B2B product. The motivation of this product was that customers in public places, such as cafés, food courts, train stations, and libraries, are unwilling to wait for seats. Under the circumstance, business may lose a lot of interests. If a sensor can be added to existing chairs, which can provide customers with instant seat usage rate and estimated waiting time, and even predict seat usage rate in the future, business can provide better services and earn more money.

In addition, these consumer behaviors can be provided to business, so that they can make more precise business strategies. For example, the café can provide products that make more profit but take more time to prepare. For example, the café can provide the light meal, when customers stay for a longer time. By contrast, the café can provide some sandwiches when consumers stay for a short time. Hence, with this censor and prediction model, it will be more convenient for customers and business can make more profit as well.


This product consists of four part: Arduino, Firebase, web page, and AI.


Arduino

The green dotted circle is the pressure sensor, which will trigger the signal when it receives force. The red solid circle is the Bluetooth transmitter. When it receives the signal from the pressure sensor, it will send the signal to the server written in Python through Bluetooth.



Firebase

When the server receives the signal from the Bluetooth sensor, the server will upload the data to Firebase and record it to provide predictive analysis input data.


Webpage

Red dots represent occupied seats, and green dots represent vacancies. The webpage gets the information from the server written in Python.



AI

This project used LSTM model to predict the vacancies.

AIoT in Industry 4.0

This is an AI website designed for people who don't know programming well enough. This website helps people to train their own AI model with easy operation and helps them solve real-world problems. I am responsible for Autoencoder application.

In addition to the basic deep Autoencoder, I also implemented other three types of Autoencoder, including convolutional Autoencoder, Deoising Autoencoder, Variational Autoencoder.

There are four applications down below, which are dimensionality reduction, image denoising, feature extraction, and anomaly detection.


Dimensionality reduction

Implemented by deep Autoencoder or convolutional Autoencoder, users can choose the model by themselves. In addition, users can also determine the parameters, including encoded dimension, activation function, optimizer, loss function, batch size, and epochs. Moreover, in the convolutional Autoencoder, users can select filter, kernel size, and pool size of each layer. The following example uses MNIST handwritten digit to reduce the original 784-dimensional image into 30-dimensional representation through the encoder, and then restore by the decoder to 784-dimensional presentation.

Object Detection

This is the program that utilize Matlab deep learning toolbox as image recognition. Labeling the objects, besides, the program is designed to position the items as well. The raw data is divided into two categories. One category contains only one object in the image while there are two objects for the other one. The program is completed in Matlab including GUI, Neural Network Training Program, Batch Process Program. The final accuracy is up to 85%.
